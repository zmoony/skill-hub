```makedown
# Git 提交信息规范（Angular 规范）
## 1. 提交格式
<type>(<scope>): <subject>

## 2. type 类型
- feat：新功能
- fix：修复bug
- docs：文档修改
- style：格式调整
- refactor：重构
- test：测试
- chore：构建/工具

## 3. scope 范围
- controller
- service
- mapper
- entity
- config
- 模块名

## 4. subject 描述
- 简洁明了
- 动词开头
- 不超过50字
- 不使用句号

## 5. 示例
feat(controller): 添加用户登录接口
fix(service): 修复用户状态更新异常
docs(readme): 更新项目部署文档
```
