
# 1. 接口测试执行提示词  
  
## 1. 测试范围  
  
对项目中**所有已注册的微服务接口**进行系统性测试，覆盖以下服务：  
  
| 服务 | 基础路径 | 说明 |  
|------|---------|------|  
| wva-gateway | `/wva-gateway/` | 网关路由、鉴权过滤 |  
| wva-system-management-service | `/wva-system-management-service/` | 字典、主题等系统管理 |  
| wva-user-center-service | `/wva-user-center-service/` | 用户认证、权限管理 |  
| wva-operation-log-service | `/wva-operation-log-service/` | 操作日志记录与查询 |  
| wva-business-service | `/wva-business-service/` | 核心业务逻辑 |  
  
**覆盖要求：**  
  
- 每个 Controller 中的所有公开接口  
- 每个 HTTP 方法（GET/POST/PUT/DELETE）  
- 所有分页查询、条件查询、详情查询接口  
- 所有 Feign 内部调用接口（如有独立暴露）  
  
---  
  
## 2. 测试类型  
  
### 2.1 功能测试  
  
| 测试项 | 说明 |  
|--------|------|  
| 正向流程 | 使用合法参数，验证接口返回预期结果 |  
| 逆向流程 | 使用非法/缺失参数，验证接口返回正确错误信息 |  
| 边界值测试 | 参数取最小值、最大值、空值、超长字符串 |  
| 业务流程串联 | 多接口组合调用，验证完整业务流程（如：登录→查询→修改→删除） |  
| 数据一致性 | 写入后立即查询，验证数据一致 |  
| 分页与排序 | 验证 pageNum、pageSize、排序参数的正确性 |  
  
### 2.2 性能测试  
  
| 测试项 | 指标要求 |  
|--------|---------|  
| 单接口响应时间 | P95 < 500ms，P99 < 1000ms |  
| 并发吞吐量 | 100并发下 QPS > 50 |  
| 稳定性测试 | 持续运行 30min，无内存泄漏、无连接池耗尽 |  
| 大数据量查询 | 10万条数据分页查询响应 < 2s |  
  
### 2.3 安全性测试  
  
| 测试项 | 说明 |  
|--------|------|  
| 未认证访问 | 不带 JWT Token 访问受保护接口，应返回 401 |  
| Token 过期 | 使用过期 Token，应返回 401 |  
| Token 伪造 | 使用篡改的 Token，应返回 401 |  
| 越权访问 | 普通用户访问管理员接口，应返回 403 |  
| SQL 注入 | 参数注入 `1' OR '1'='1`，应被拦截或参数化处理 |  
| XSS 注入 | 参数注入 `<script>alert(1)</script>`，应被过滤 |  
| 敏感数据泄露 | 验证返回结果不含密码、密钥等敏感字段 |  
| IDOR（越权ID） | 使用其他用户的 ID 访问资源，应返回 403 |  
  
### 2.4 异常与容错测试  
  
| 测试项 | 说明 |  
|--------|------|  
| 服务降级 | 某微服务宕机后，网关返回友好错误而非 500 |  
| 重复请求 | 同一创建请求重复发送，验证幂等性 |  
| 超大请求体 | 发送超长 JSON body，验证 413 或截断处理 |  
| 请求超时 | 模拟慢查询，验证超时机制 |  
| 数据库异常 | 模拟 DB 连接断开，验证错误返回格式 |

## 4. 输入参数验证  
  
对每个接口的每个参数执行以下验证矩阵：  
  
| 参数类型 | 测试用例 |  
|---------|---------|  
| 必填参数 | 缺失时返回 400 + 明确错误提示 |  
| 字符串参数 | 空字符串、超长字符串、特殊字符、Unicode |  
| 数字参数 | 0、负数、超大数、非数字字符串 |  
| 日期参数 | 过去日期、未来日期、非法格式、空值 |  
| 枚举参数 | 合法值、非法值、空值 |  
| ID 参数 | 存在的 ID、不存在的 ID、其他用户的 ID |  
| 分页参数 | pageNum=0、pageNum=-1、pageSize=0、pageSize=99999 |  
| JSON Body | 缺少字段、多余字段、类型错误、嵌套对象异常 |  
  
---  
  
## 5. 返回结果校验  
  
每个接口返回结果需校验以下维度：  
  
| 校验项 | 规则 |  
|--------|------|  
| HTTP 状态码 | 成功 200/201，客户端错误 400/401/403，服务端错误 500 |  
| 响应格式 | 必须符合项目统一响应体格式：`{code, message, data}` |  
| code 字段 | 成功=200（或约定值），错误=对应错误码 |  
| message 字段 | 成功时为"操作成功"类信息，错误时为具体错误描述 |  
| data 字段 | 成功时包含业务数据，失败时为 null 或空 |  
| 字段类型 | 每个返回字段类型与 API 文档定义一致 |  
| 字段完整性 | 返回字段与文档定义完全匹配，无遗漏无多余 |  
| 分页响应 | 包含 total、pageNum、pageSize、records 结构 |  
| 时间格式 | 统一为 `yyyy-MM-dd HH:mm:ss` |  
| 空值处理 | null 字段不返回或显式返回 null，保持一致 |  
  
---  
  
## 6. 错误处理机制验证  
  
| 错误场景 | 验证要点 |  
|---------|---------|  
| 参数校验失败 | 返回 400，message 列出所有校验失败字段 |  
| 未认证 | 返回 401，message="Token无效或已过期" |  
| 无权限 | 返回 403，message="无操作权限" |  
| 资源不存在 | 返回 404，message="资源不存在" |  
| 业务规则冲突 | 返回 200 + code=业务错误码，如重复创建 |  
| 服务内部异常 | 返回 500，message 不暴露堆栈信息 |  
| 网关路由失败 | 返回 503，message="服务不可用" |  
| 限流触发 | 返回 429，message="请求过于频繁" |  
  
**关键验证：所有错误响应绝不暴露堆栈信息、SQL 语句、内部 IP 等敏感信息。**

### 自动化测试脚本示例（RestAssured）  
  
```java  
@Test  
void testCreateDictType() {  
    String token = loginAndGetToken("test_admin", "password");  
    given()        .header("Authorization", "Bearer " + token)        .contentType(ContentType.JSON)        .body("{\"code\":\"TEST_001\",\"name\":\"测试字典\",\"status\":1}")  
    .when()        .post("/wva-system-management-service/dict-type")    .then()        .statusCode(201)        .body("code", equalTo(200))        .body("message", equalTo("创建成功"))  
        .body("data.id", notNullValue());}  
```
